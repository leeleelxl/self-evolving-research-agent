## 系统架构

```
用户输入研究问题
       │
       ▼
┌─────────────┐
│   Planner   │ ← 问题分解 + 检索策略规划
└──────┬──────┘
       │ 子问题列表
       ▼
┌─────────────┐     ┌──────────────┐
│  Retriever  │────▶│ 知识库(FAISS) │
│             │◀────│ + BM25 Index  │
└──────┬──────┘     └──────────────┘
       │ 相关论文
       ▼
┌─────────────┐
│   Reader    │ ← 论文精读 + 信息提取
└──────┬──────┘
       │ 结构化笔记
       ▼
┌─────────────┐
│   Writer    │ ← 综述生成
└──────┬──────┘
       │ 综述草稿
       ▼
┌─────────────┐
│   Critic    │ ← Agent-as-Judge 评估
└──────┬──────┘
       │ 评分 + 反馈
       ▼
  反馈是否达标？
   ├── 是 → 输出最终综述
   └── 否 → 反馈回 Planner（自进化循环）
```

## Agent 职责

| Agent | 输入 | 输出 | 核心能力 |
|-------|------|------|---------|
| Planner | 研究问题 + Critic 反馈 | 子问题 + 检索策略 | 问题分解、策略调整 |
| Retriever | 子问题 + 策略 | 相关论文列表 | Semantic Scholar/arXiv API、混合检索 |
| Reader | 论文 PDF/摘要 | 结构化笔记 | 信息提取、关键发现总结 |
| Writer | 结构化笔记集合 | 综述文档 | 学术写作、引用管理 |
| Critic | 综述文档 | 评分 + 改进建议 | 质量评估、覆盖度检查 |

## 自进化机制

1. Critic 对综述评分（覆盖度、深度、准确性、连贯性）
2. 低分维度生成具体改进建议
3. 建议反馈给 Planner，调整检索策略
4. 新一轮检索-阅读-写作
5. 记录每轮的策略变化和效果变化（用于消融实验）
