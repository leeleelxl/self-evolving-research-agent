# Session 记录 — 2026-04-08

## 本次完成的所有工作

### 代码（15 个核心模块）
- `core/models.py` — 10+ Pydantic 数据模型（Paper, Chunk, PaperNote, ResearchPlan, CriticFeedback 等）
- `core/config.py` — 参数化配置，消融变量 A(chunk) B(retrieval) C(rerank)
- `core/llm.py` — 双后端 LLM（OpenAI 中转站 + Anthropic），含 `_dereference_schema` 解决 $defs 兼容问题
- `core/agent.py` — Agent 抽象基类，用 `create_llm_client` 工厂函数
- `agents/planner.py` — 问题分解 + 迭代轮策略调整（注入 CriticFeedback）
- `agents/retriever.py` — asyncio.Semaphore 并发检索 + 去重（paper_id + 标题归一化）
- `agents/reader.py` — 并发精读 + relevance_score 过滤（<0.3 丢弃）
- `agents/writer.py` — 综合 PaperNote 按 sub_questions 生成 ResearchReport
- `agents/critic.py` — 4 维评分 + 代码层面重算 is_satisfactory（不信 LLM 判断）
- `retrieval/search.py` — Semantic Scholar + arXiv 客户端（httpx async）
- `retrieval/chunking.py` — 3 种策略：fixed / semantic / recursive
- `retrieval/indexing.py` — DenseIndex(numpy) + SparseIndex(BM25) + HybridIndex(RRF)
- `retrieval/reranker.py` — LLM reranker，失败时 fallback 0.5
- `retrieval/embedding.py` — fastembed 封装（BGE-small-en, 384 维）
- `pipeline/research.py` — 5 Agent 编排 + 自进化循环 + 跨轮去重

### 踩过的坑
1. OpenAI API 不支持 Pydantic 的 `$defs`/`allOf` → 写了 `_dereference_schema` 内联展开
2. `tuple[int, int] | None` schema 不兼容 OpenAI → 改为两个独立字段 `year_min`/`year_max`
3. arXiv HTTP→HTTPS 迁移 → `follow_redirects=True`
4. 中转站只支持流式（非流式返回空 content）→ 全部用流式 + 累积
5. Semantic Scholar 无 key 时 IP 级别限速严格 → 429 时 skip 不 fail
6. fastembed 首次加载需下载模型（~50MB）
